# 最小业务资源模板

> 目标：落实 `018_脚手架复用适用性审查` 的 P1 路线，先用文档模板降低新增业务资源的重复劳动，不急于做代码生成器。本文以 `projects`（项目档案）为例，展示一个后台资源从数据库、权限、模型、路由、控制器、菜单、测试到文档同步的最小闭环。

## 1. 使用原则

这个模板用于复制到具体业务项目中改造，不建议直接把 `projects` 放入本脚手架内核。

原因：

- `projects` 是示例业务资源，不是所有后台项目都需要。
- 脚手架内核应保持认证、用户、RBAC、审计、数据库初始化等通用能力。
- 示例模板能提高复用效率，同时避免把内核做成大而全业务系统。

推荐使用方式：

```text
1. 复制本文模板。
2. 把 projects 替换为真实资源名，例如 products、orders、devices。
3. 按项目实际字段裁剪。
4. 同步 DDL、schema、seed、菜单、测试和接口文档。
5. 跑 SQLite 全量测试；如果使用 MySQL，跑 test:mysql。
```

## 4. SQLite DDL 模板

写入 `src/db/sqlite-schema.sql`：

```sql
CREATE TABLE IF NOT EXISTS projects (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  code TEXT NOT NULL UNIQUE,
  description TEXT,
  status TEXT DEFAULT 'active',
  created_at INTEGER DEFAULT (strftime('%s','now')),
  updated_at INTEGER DEFAULT (strftime('%s','now')),
  deleted_at INTEGER
);

CREATE INDEX IF NOT EXISTS idx_projects_code ON projects(code);
CREATE INDEX IF NOT EXISTS idx_projects_status ON projects(status);
CREATE INDEX IF NOT EXISTS idx_projects_deleted_at ON projects(deleted_at);
CREATE INDEX IF NOT EXISTS idx_projects_created_at ON projects(created_at);
```

要求：

- `code` 作为业务唯一标识，便于导入和外部系统对接。
- `deleted_at` 用于软删除。
- 列表常用筛选字段必须有索引。

## 7. Model 模板

文件：`src/models/ProjectModel.js`

```js
import BaseModel from './BaseModel.js'
import { projects } from '../models/schema/index.js'
import { getCurrentTimestamp } from '../utils/datetime.js'
import AppError from '../utils/AppError.js'

class ProjectModel extends BaseModel {
  constructor() {
    super('projects', projects)
    this.searchableFields = ['name', 'code', 'description']
    this.safeFields = ['id', 'name', 'code', 'description', 'status', 'created_at', 'updated_at']
    this.safeOrderFields = ['id', 'name', 'code', 'status', 'created_at', 'updated_at']
  }

  async beforeCreate(data) {
    if (!data.name || data.name.trim() === '') {
      throw new AppError('项目名称不能为空', 400, 'PROJECT_NAME_REQUIRED')
    }

    if (!data.code || data.code.trim() === '') {
      throw new AppError('项目编码不能为空', 400, 'PROJECT_CODE_REQUIRED')
    }

    const isCodeUnique = await this.isUnique('code', data.code)
    if (!isCodeUnique) {
      throw new AppError('项目编码已存在', 409, 'PROJECT_CODE_EXISTS')
    }

    return {
      ...data,
      status: data.status || 'active'
    }
  }

  async getProjectList(filter = {}, page = 1, limit = 20) {
    const queryFilter = { ...filter, deleted_at: null }
    const total = await this.count(queryFilter)
    const rows = await this.findPage(queryFilter, {
      page,
      limit,
      orderBy: 'created_at',
      order: 'desc',
      fields: this.safeFields
    })

    return {
      projects: rows,
      pagination: {
        page,
        limit,
        total,
        pages: Math.ceil(total / limit)
      }
    }
  }

  async softDeleteProject(id) {
    return await this.update(id, { deleted_at: getCurrentTimestamp() })
  }
}

export default ProjectModel
```

注意：

- 不要把 HTTP 请求对象传入 Model。
- 列表分页必须下推数据库。
- `safeFields` 不包含内部敏感字段。
- 当前 `BaseModel.findPage()` 不接收 `search` 参数；需要搜索时应参考 `UserModel.getUserList()` 用 Drizzle `LIKE` 条件构造 `whereClause`，不要先查全量再内存切片。

## 8. Controller 模板

文件：`src/controllers/ProjectController.js`

```js
import { projectModel, auditLogModel } from '../models/index.js'
import { asyncHandler } from '../middleware/error/index.js'
import AppError from '../utils/AppError.js'
import * as logger from '../utils/logger/index.js'

function getRequestUserId(c) {
  return c.get('user')?.id || null
}

async function recordProjectAudit(c, entry) {
  await auditLogModel.record({
    userId: getRequestUserId(c),
    ipAddress: c.req.header('x-forwarded-for') || c.req.header('x-real-ip') || 'unknown',
    userAgent: c.req.header('user-agent') || 'unknown',
    ...entry
  })
}

class ProjectController {
  getProjects = asyncHandler(async (c) => {
    const query = c.get('validatedQuery') || {}
    const page = query.page || 1
    const limit = query.limit || 20
    const filter = {}
    if (query.status) filter.status = query.status

    const result = await projectModel.getProjectList(filter, page, limit)
    return c.json({ success: true, data: result, meta: { timestamp: new Date().toISOString() } })
  })

  getProjectById = asyncHandler(async (c) => {
    const { id } = c.get('validatedParams')
    const project = await projectModel.findById(id)
    if (!project || project.deleted_at) {
      throw new AppError('项目不存在', 404, 'PROJECT_NOT_FOUND')
    }
    return c.json({ success: true, data: project, meta: { timestamp: new Date().toISOString() } })
  })

  createProject = asyncHandler(async (c) => {
    const body = c.get('validatedBody')
    const project = await projectModel.create(body)

    logger.business('project_created', { user_id: getRequestUserId(c), project_id: project.id })
    await recordProjectAudit(c, {
      action: 'projects.create',
      resourceType: 'projects',
      resourceId: project.id,
      newValues: project
    })

    return c.json({ success: true, data: project, meta: { timestamp: new Date().toISOString() } }, 201)
  })
}

export const projectController = new ProjectController()
export default projectController
```

注意：

- 模板只展示最小闭环，更新、删除、状态切换按同样模式补齐。
- 第三个业务 Controller 出现后，可以再抽 `recordRequestAudit()`，当前不提前抽象。

## 9. Route 与 schema 模板

文件：`src/routes/project-schemas.js`

```js
export const projectListQuerySchema = {
  type: 'object',
  properties: {
    page: { type: 'integer', minimum: 1, default: 1 },
    limit: { type: 'integer', minimum: 1, maximum: 100, default: 20 },
    status: { type: 'string', enum: ['active', 'inactive'] },
    search: { type: 'string', minLength: 1, maxLength: 100 }
  },
  additionalProperties: false
}

export const createProjectSchema = {
  type: 'object',
  properties: {
    name: { type: 'string', minLength: 1, maxLength: 100 },
    code: { type: 'string', minLength: 1, maxLength: 100 },
    description: { type: 'string', maxLength: 1000 },
    status: { type: 'string', enum: ['active', 'inactive'] }
  },
  required: ['name', 'code'],
  additionalProperties: false
}
```

文件：`src/routes/projects.js`

```js
import { Hono } from 'hono'
import { projectController } from '../controllers/ProjectController.js'
import { authMiddleware, requirePermissions } from '../middleware/auth.js'
import { validateBody, validateParams, validateQuery, schemas } from '../middleware/validation/index.js'
import { createProjectSchema, projectListQuerySchema } from './project-schemas.js'

const projectRoutes = new Hono()

projectRoutes.use('*', authMiddleware)

projectRoutes.get('/', requirePermissions('projects:list'), validateQuery(projectListQuerySchema), projectController.getProjects)
projectRoutes.post('/', requirePermissions('projects:create'), validateBody(createProjectSchema), projectController.createProject)
projectRoutes.get('/:id', requirePermissions('projects:read'), validateParams(schemas.id), projectController.getProjectById)

export default projectRoutes
```

注意：

- 如果当前项目没有 `requirePermissions` 中间件，应先补权限中间件，或使用已有 RBAC 模型实现，不要只做前端权限。
- 静态路由必须放在动态 `/:id` 前面。
- 控制器读取 `validated*`，不要重复解析原始请求。

## 14. 不做生成器的原因

当前先不做 `bun run scaffold:resource projects`，原因：

- 还没有在 2-3 个真实业务资源中验证模板稳定性。
- 过早生成器会固化错误抽象。
- 文档模板足以减少第一轮重复劳动。
- 生成器应在模板被多项目复用后再实现。

触发生成器的条件：

```text
1. 至少 2 个真实业务项目复用了该模板。
2. 模板字段和目录结构基本稳定。
3. 手工复制仍然成为明显成本。
4. 有测试能验证生成后的资源可启动。
```
