# MySQL 初始化与种子数据

> 目标：补齐后台脚手架在不集成 Logto 时的数据库复用能力，让 SQLite 与 MySQL 在核心表、默认 RBAC、系统配置和管理员初始化上保持同源、幂等、可验证。

## 1. 设计结论

当前 MySQL 支持采用轻量脚本化方案：

- `src/db/mysql-schema.js` 保存 MySQL 幂等 DDL 与索引初始化语句。
- `src/db/seed.js` 复用同一套默认角色、权限、角色授权、默认管理员和系统配置常量。
- `src/db/index.js` 在 `DATABASE_TYPE=mysql` 时默认执行建表和 seed。
- `DB_AUTO_MIGRATE=false` 可以关闭自动建表。
- `DB_SEED_DEFAULTS=false` 可以关闭默认 seed。

没有引入独立迁移框架，是因为当前目标是小而美的后台脚手架：先保证多项目低成本落地和初始化闭环；等需要版本化迁移、回滚、灰度时，再引入 Drizzle Kit 或专用迁移工具。

## 2. 初始化流程

```text
1. 设置 DATABASE_TYPE=mysql。
2. 通过 DATABASE_URL 或 DB_HOST/DB_PORT/DB_USER/DB_PASSWORD/DB_NAME 配置连接。
3. initDb() 创建 mysql2 pool。
4. 默认调用 applyMysqlSchema(pool) 执行 CREATE TABLE IF NOT EXISTS。
5. 默认调用 seedMysqlDefaults(pool) 写入角色、权限、角色权限、管理员和系统配置。
6. 创建 Drizzle MySQL 实例供模型层使用。
```

关键环境变量：

| 变量 | 说明 | 默认值 |
| --- | --- | --- |
| `DATABASE_TYPE` | 数据库类型，MySQL 时设为 `mysql` | `sqlite` |
| `DATABASE_URL` | MySQL 连接 URL，优先级最高 | 无 |
| `DB_HOST` | MySQL 主机 | `localhost` |
| `DB_PORT` | MySQL 端口 | `3306` |
| `DB_USER` | MySQL 用户 | `root` |
| `DB_PASSWORD` | MySQL 密码 | 空 |
| `DB_NAME` | MySQL 数据库名 | `bun_server` |
| `DB_AUTO_MIGRATE` | 是否自动执行 DDL，设为 `false` 关闭 | 开启 |
| `DB_SEED_DEFAULTS` | 是否自动写入默认 seed，设为 `false` 关闭 | 开启 |

## 3. 默认 seed parity

MySQL 与 SQLite 共享以下数据源：

- `DEFAULT_ROLES`
- `DEFAULT_PERMISSIONS`
- `DEFAULT_ROLE_PERMISSIONS`
- `DEFAULT_ADMIN_USER`
- `DEFAULT_SYSTEM_CONFIGS`

MySQL seed 使用 `ON DUPLICATE KEY UPDATE`，SQLite seed 使用 `ON CONFLICT ... DO UPDATE`。两者都要求可重复执行，不因重复初始化造成脏数据或唯一键冲突。

## 4. 验证方式

不依赖真实 MySQL 的默认验证：

```text
bun run tests/test-mysql-schema.js
bun run tests/test-mysql-init-plan.js
```

真实连接级验证：

```text
MYSQL_TEST_DATABASE_URL="mysql://user:pass@127.0.0.1:3306/bun_server_test" bun run test:mysql
```

注意：真实连接级测试会在目标库执行建表和 seed，必须指向测试库或临时库，不要指向生产库。

## 5. 当前边界

已完成：

- MySQL 核心表 DDL。
- MySQL 核心索引初始化。
- MySQL 默认 RBAC 与系统配置 seed parity。
- 无 MySQL 服务时的初始化计划测试。
- 有 MySQL 服务时的真实连接级测试脚本。

暂不做：

- 版本化迁移历史表。
- 回滚脚本。
- 在线迁移灰度策略。
- 生产数据备份恢复策略。

这些能力属于生产运维体系，不应默认塞进轻量后台脚手架核心。

## 6. 踩坑记录

### 6.1 schema parity 不等于初始化可用

背景：此前 `src/models/schema/mysql.js` 已导出核心 Drizzle 表，但不能证明 MySQL 能完成建表、索引和 seed。

原因：Drizzle schema 是运行时模型映射，不等同于数据库初始化脚本；没有真实 DDL/seed 路径时，新项目接入 MySQL 仍要重复手写初始化。

排查过程：检查 `src/db/index.js` 后发现 SQLite 会执行 `sqlite-schema.sql` 和 `seedSqliteDefaults()`，MySQL 只创建连接池和 Drizzle 实例。

解决方案：新增 `src/db/mysql-schema.js`、`seedMysqlDefaults()` 和 `tests/test-mysql-init-plan.js`，并在 MySQL 初始化路径中默认执行。

建议：后续每次新增核心表或默认 seed，都必须同步更新 SQLite DDL、MySQL DDL、Drizzle schema、seed、测试和文档。
